How Memori Works

üéØ What is Memori?

Memori is a memory layer for AI applications that automatically captures facts from conversations and persists them to a database. This enables AI systems to remember information across sessions and provide contextually aware responses.

üß† Automatic Memory Capture

When you interact with the AI, Memori automatically extracts factual information from the conversation. For example, if you say "My favorite color is blue," Memori stores this fact in the database without any explicit programming needed. 

User: "My name is Sarah and I work as a software engineer"
AI: [Responds naturally]
Memori: [Automatically stores: "Sarah is a software engineer"]

üß≠ With Memori vs. Without

See how persistence changes the experience:

Without Memori (stateless):
User: "My name is Maya and I live in Seattle."
AI: "Nice to meet you, Maya!"
User (later): "Where do I live?"
AI: "I don't have that info."

With Memori (stateful):
User: "My name is Maya and I live in Seattle."
AI: "Great to meet you, Maya!"
User (later): "Where do I live?"
AI: "You live in Seattle."
  • Context carry-over: Prior facts are recalled automatically.
  • Isolation: Each attendee/session has its own memory.
  • Resettable: Use the reset button in the chat to start fresh.

üîç Memory Recall

In subsequent conversations, Memori retrieves relevant stored facts and provides them to the AI model as context. This allows the AI to answer questions about previously shared information. 

User: "What's my name?"
AI: [Receives context from Memori]
AI: "Your name is Sarah!"

üîê Attribution & Isolation

Memori uses entity_id and process_id to isolate memories per user or application. This ensures that:

  • Each user's memories are kept separate
  • Different applications can use the same database without conflicts
  • Privacy and data isolation are maintained

üíæ Storage Backends

Memori supports multiple storage backends:

  • SQLite - Used in this demo, perfect for development and small applications
  • PostgreSQL - Production-ready for larger applications
  • Other databases - Via SQLAlchemy

‚ö° How It Works Under the Hood

  1. LLM Call: Your message is sent to the AI model (OpenAI or Gemini)
  2. Response: The AI generates a natural response
  3. Augmentation: Memori asynchronously extracts facts from the conversation
  4. Storage: Facts are stored in the database with attribution
  5. Recall: On future queries, relevant facts are retrieved and provided as context
